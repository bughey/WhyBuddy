# 宠物角色模型规格书

## 概述

本文档定义 Cube Pets 宠物角色系统中两种基础体型的模型规格。所有宠物角色共享统一骨骼 `SK_CubePet_Skeleton`（29 骨骼），通过 Skeletal Mesh 切换外观。

- **适用引擎**：Unreal Engine 5
- **统一骨骼**：`SK_CubePet_Skeleton`
- **单角色面数上限**：≤ 5000 三角面
- **坐标系**：UE5 左手坐标系（前方 +X，上方 +Z）
- **单位**：厘米（1 UE 单位 = 1 cm）

---

## 体型定义

### Type A：Compact 猫型（Cat-like）

| 属性     | 值                                            |
| -------- | --------------------------------------------- |
| 资产名称 | `SM_CubePet_Cat`                              |
| 设计风格 | 短小圆润、大头小身、四肢短粗，整体偏 Q 版可爱 |
| 适用角色 | 灵巧型、辅助型宠物                            |

### Type B：Standard 犬型（Dog-like）

| 属性     | 值                                             |
| -------- | ---------------------------------------------- |
| 资产名称 | `SM_CubePet_Dog`                               |
| 设计风格 | 身体较高、四肢修长、头身比适中，整体偏活泼精干 |
| 适用角色 | 执行型、主力型宠物                             |

## 骨骼绑定要求

### 统一骨骼：SK_CubePet_Skeleton

两种体型必须绑定到同一套骨骼 `SK_CubePet_Skeleton`（29 骨骼）。骨骼层级结构详见 `docs/ue-character/skeleton-hierarchy.md`。

### 蒙皮权重要点

| 要求                 | 说明                                                      |
| -------------------- | --------------------------------------------------------- |
| 每顶点最大骨骼影响数 | ≤ 4                                                       |
| 对称性               | 左右对称部位的权重分布应镜像一致                          |
| 头部权重             | Head 骨骼为主权重；Eye_L / Eye_R / Jaw 仅影响面部表情区域 |
| 躯干权重             | Pelvis → Spine_01 → Spine_02 平滑过渡，避免弯腰时出现折叠 |
| 四肢权重             | 关节处（肘、膝）需要 2-3 根骨骼混合权重，避免变形塌陷     |
| 手指权重             | Index_01 和 Thumb_01 各自独立控制，Hand 为基础权重        |
| 脚趾权重             | Toe 骨骼独立控制脚趾弯曲，Foot 为基础权重                 |

### 体型差异的骨骼适配

由于两种体型共享同一骨骼，骨骼的相对位置在绑定时需要按体型调整：

- **Cat 型**：骨骼间距更紧凑，Pelvis 位置较低，四肢骨骼链较短
- **Dog 型**：骨骼间距更舒展，Pelvis 位置较高，四肢骨骼链较长
- 骨骼的旋转轴方向必须保持一致，确保动画资产可在两种体型间复用

## Morph Target 表情支持

两种体型均必须支持以下 6 个 Morph Target，通过顶点偏移实现面部表情变化：

| 表情 | Morph Target 名称 | 主要变形区域           | 说明                                 |
| ---- | ----------------- | ---------------------- | ------------------------------------ |
| 中性 | `Expr_Neutral`    | —                      | 默认状态，所有 Morph Target 权重为 0 |
| 开心 | `Expr_Happy`      | 嘴角上扬、眼睛微眯     | 眉毛略抬，整体表情放松愉悦           |
| 悲伤 | `Expr_Sad`        | 嘴角下垂、眉毛下压     | 眼睛略闭合，整体表情低落             |
| 思考 | `Expr_Thinking`   | 单侧眉毛上扬、嘴巴微偏 | 头部可配合轻微歪斜动画               |
| 惊讶 | `Expr_Surprised`  | 眼睛睁大、嘴巴张开     | 眉毛大幅上扬                         |
| 生气 | `Expr_Angry`      | 眉毛下压内收、嘴巴紧闭 | 可配合轻微颤抖动画                   |

### Morph Target 制作要求

- 每个 Morph Target 的顶点偏移量应控制在合理范围内，避免穿模
- 多个表情可叠加使用（如 work + thinking），叠加时不应出现严重变形
- 面部表情区域的顶点密度应足够支撑表情变化的平滑度
- Cat 型由于头部占比更大，面部表情区域可分配更多顶点
- Dog 型面部表情应保持与 Cat 型一致的表情幅度感知

## UV 布局规范

### 通用要求

| 参数          | 值                                                        |
| ------------- | --------------------------------------------------------- |
| UV 集数量     | 1（单 UV 集）                                             |
| UV 空间利用率 | ≥ 85%                                                     |
| 纹理分辨率    | 1024 × 1024（LOD0）；512 × 512（LOD1）；256 × 256（LOD2） |
| 纹理格式      | PNG（制作阶段）；BC7 / DXT5（引擎内压缩）                 |
| 材质槽数量    | 1（单材质槽，通过材质参数切换外观变体）                   |

### UV 区域分配建议

| 区域        | UV 空间占比 | 说明                           |
| ----------- | ----------- | ------------------------------ |
| 头部 + 面部 | 35% - 40%   | 面部细节最多，需要最高纹素密度 |
| 躯干        | 25% - 30%   | 身体主体区域                   |
| 四肢        | 20% - 25%   | 手臂 + 腿部                    |
| 配饰区域    | 5% - 10%    | 预留给可选配件的 UV 空间       |

### UV 制作规范

- UV 接缝应尽量放在不可见区域（腋下、腿内侧、后脑勺）
- 面部区域不允许有 UV 接缝穿过
- 左右对称部位可共享 UV 空间（镜像 UV）以提高纹素利用率
- UV 岛之间保留至少 4 像素间距（基于 1024 分辨率），避免 Mipmap 渗色

## LOD（Level of Detail）规范

### LOD 层级定义

| LOD 层级 | 三角面比例 | Cat 型面数 | Dog 型面数 | 切换距离       | 说明                     |
| -------- | ---------- | ---------- | ---------- | -------------- | ------------------------ |
| LOD0     | 100%       | 4500       | 4800       | 0 - 1500 cm    | 完整模型，所有细节保留   |
| LOD1     | 50%        | 2250       | 2400       | 1500 - 3000 cm | 简化手指、脚趾、面部细节 |
| LOD2     | 25%        | 1125       | 1200       | 3000 cm+       | 大幅简化，仅保留轮廓     |

### LOD 简化策略

#### LOD1（50% 简化）

- 手指合并为单一手掌形状（Index_01 + Thumb_01 区域合并）
- 脚趾合并为单一脚掌形状
- 面部细节简化（眼睛、嘴巴减面，但保留 Morph Target 基本功能）
- 耳朵简化为低面数版本
- 配饰区域面数减半

#### LOD2（25% 简化）

- 四肢简化为圆柱体近似
- 面部仅保留基本轮廓，Morph Target 可禁用
- 手指、脚趾完全合并
- 配饰不渲染
- 可切换为更简单的材质（去除法线贴图）

### LOD 骨骼简化

| LOD 层级 | 可移除的骨骼                                                 | 说明                                                |
| -------- | ------------------------------------------------------------ | --------------------------------------------------- |
| LOD0     | 无                                                           | 使用完整 29 骨骼                                    |
| LOD1     | Index_01_L, Index_01_R, Thumb_01_L, Thumb_01_R, Toe_L, Toe_R | 移除手指和脚趾骨骼（6 骨骼），剩余 23 骨骼          |
| LOD2     | 同 LOD1 + Eye_L, Eye_R, Jaw                                  | 进一步移除面部骨骼（累计移除 9 骨骼），剩余 20 骨骼 |

## 验证清单

模型制作完成后，需通过以下验证项：

### 骨骼与绑定

- [ ] 两种体型均成功绑定到 `SK_CubePet_Skeleton`
- [ ] 骨骼层级与 `skeleton-hierarchy.md` 定义完全一致（29 骨骼）
- [ ] Root 骨骼位于模型脚底中心 (0, 0, 0)
- [ ] Pelvis 位于角色重心位置
- [ ] 左右对称骨骼权重镜像一致

### 面数与性能

- [ ] Cat 型 LOD0 ≤ 4500 三角面
- [ ] Dog 型 LOD0 ≤ 4800 三角面
- [ ] 两种体型均不超过 5000 三角面上限
- [ ] 每顶点骨骼影响数 ≤ 4
- [ ] LOD1 面数约为 LOD0 的 50%
- [ ] LOD2 面数约为 LOD0 的 25%

### Morph Target

- [ ] 6 个 Morph Target 均正常工作（Neutral / Happy / Sad / Thinking / Surprised / Angry）
- [ ] 多表情叠加无严重穿模或变形
- [ ] Cat 型与 Dog 型的表情幅度感知一致

### UV 与纹理

- [ ] 单 UV 集，UV 空间利用率 ≥ 85%
- [ ] 面部区域无 UV 接缝
- [ ] UV 岛间距 ≥ 4 像素（1024 分辨率基准）
- [ ] 纹理分辨率符合规范（1024 / 512 / 256）

### 动画兼容性

- [ ] 同一动画序列可在 Cat 型和 Dog 型之间复用，无明显穿模
- [ ] IK 链（手臂 / 腿部 / 脊柱）在两种体型上均正常工作
- [ ] 动画过渡时无 T-Pose 闪现
