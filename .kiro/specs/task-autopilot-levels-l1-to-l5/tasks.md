# 任务清单：任务自动驾驶 L1-L5 分级

- [ ] 补齐 `L1-L5` 的统一术语表，确保与 `task-autopilot-core-concepts` 中的 `Destination / Route / Drive State / Fleet / Takeover` 保持一致。
- [ ] 在 steering 层补一份自动驾驶分级摘要，统一 README、项目总览、主线规划文档的等级口径。
- [ ] 为 `L1` 定义“路线建议级”的最小产品行为，包括任务发起、路线推荐、用户确认与人工启动执行的边界。
- [ ] 为 `L2` 定义“部分自动执行级”的最小产品行为，包括低风险自动推进与关键节点接管的边界。
- [ ] 为 `L3` 定义“标准任务自动闭环级”的最小产品行为，包括标准任务范围、自动 review / revise、异常降级与交付策略。
- [ ] 为 `L4` 定义“限定任务域高自动化级”的白名单约束模型，包括任务域白名单、权限白名单、预算白名单与策略版本。
- [ ] 明确 `L5` 仅作为远期研究目标，统一禁止在 README、架构图、项目总览和产品文案中把当前能力写成 `L5` 已实现。
- [ ] 设计任务级 `autopilotLevel` 与运行时 `effectiveAutopilotLevel` 的元数据模型，明确其在 `Mission`、`Workflow`、runtime context 中的挂载位置。
- [ ] 设计自动驾驶等级与十阶段工作流阶段的映射表，明确每个阶段在 `L1-L4` 下允许的自动化动作范围。
- [ ] 设计自动驾驶等级与 Web-AIGC 节点族 / route family 的映射表，至少覆盖导航节点、执行节点、治理节点三类。
- [ ] 设计自动驾驶等级与 runtime governance 的对接方案，明确预算、权限、风险动作、外部副作用对等级降级或接管的触发条件。
- [ ] 设计自动驾驶等级与 HITL / DecisionPanel 的对接方案，明确澄清、路线确认、风险审批、交付确认四类接管点。
- [ ] 在驾驶舱信息架构中加入“目标等级 / 当前等级 / 降级原因 / 接管原因”的展示位置与展示语义。
- [ ] 为 replay / audit / observability 设计等级证据字段，包括等级宣告、等级变更、接管原因、降级原因与交付确认信息。
- [ ] 设计等级变更时间线，支持在回放与审计面板中查看任务从 `L3` 降级到 `L2` 等关键事件。
- [ ] 为标准任务场景制定首批试点清单，明确哪些任务可以先按 `L2` 或 `L3` 推进，哪些任务必须停留在 `L1`。
- [ ] 为高风险动作制定首批“必须接管”清单，至少覆盖外部写操作、向量删除、宿主打开动作、权限升级、成本越界等场景。
- [ ] 补一份等级验收矩阵，按“任务类型 × 风险等级 × 自动驾驶等级 × 接管要求”统一验收口径。
- [ ] 规划前后端实现顺序，先完成等级定义与展示，再推进 runtime metadata 与治理接线，最后扩展到白名单高自动化场景。
