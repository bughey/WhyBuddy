# 需求文档：响应式适配与回归收尾

## 简介

本规格是 UI 改造项目（共 8 个 spec）的第 8 个，也是最后一个，依赖前 7 个 spec 全部完成。目标是对整个 UI 改造进行响应式适配验证、断点回归、视觉一致性收尾和跨浏览器兼容性检查，确保改造后的界面在桌面端（1280px+）、平板端（768–1279px）、移动端（≤767px）三种视口下均能正常工作。

## 需求

### 需求 1：桌面端断点回归

**用户故事：** 作为桌面端用户，我希望在 1280px、1440px、1728px 三个常见桌面宽度下界面布局正确、无溢出、无重叠。

#### 验收标准

1. THE UI SHALL 在 1280px 宽度下正确渲染：左侧 240px 侧边栏 + 中央内容区 + 右侧 300–360px 信息面板
2. THE UI SHALL 在 1440px 宽度下正确渲染，中央内容区自然扩展
3. THE UI SHALL 在 1728px 宽度下正确渲染，无水平溢出
4. THE UI SHALL 在所有桌面宽度下保持卡片间距、字体大小、圆角和阴影的视觉一致性

### 需求 2：平板端断点回归

**用户故事：** 作为平板端用户，我希望在 768px–1279px 范围内界面布局正确，侧边栏折叠为图标模式。

#### 验收标准

1. THE UI SHALL 在 1024px 宽度下正确渲染：左侧 64px 图标侧边栏 + 中央内容区 + 右侧信息面板
2. THE UI SHALL 在 768px 宽度下正确渲染，右侧信息面板可能折叠或隐藏
3. THE UI SHALL 在平板端保持所有交互功能可用（导航、任务选择、发起面板、状态指示器）

### 需求 3：移动端断点回归

**用户故事：** 作为移动端用户，我希望在 375px–767px 范围内界面布局正确，使用底部标签栏导航。

#### 验收标准

1. THE UI SHALL 在 375px 宽度下正确渲染：无侧边栏、底部标签栏、全宽内容区
2. THE UI SHALL 在 414px 宽度下正确渲染
3. THE UI SHALL 在移动端保持核心功能可用（导航、任务查看、任务发起）
4. THE UI SHALL 在移动端尊重 `env(safe-area-inset-bottom)` 安全区

### 需求 4：视觉一致性审查

**用户故事：** 作为前端开发者，我希望确认所有组件都正确消费了 spec 1 的设计令牌，没有残留的暖色硬编码值。

#### 验收标准

1. THE UI SHALL 不包含任何 `#4a3727`、`#7d6856`、`#c98257`、`rgba(88,61,39,…)` 等暖棕色硬编码值（LoadingScreen 除外，已在之前单独处理）
2. THE UI SHALL 在所有页面（首页、任务页、任务详情页、Debug 页、Replay 页）保持一致的冷灰色板
3. THE UI SHALL 确保所有 shadcn/ui 组件（Button、Dialog、Tabs、DropdownMenu 等）正确继承新令牌

### 需求 5：构建与类型检查

**用户故事：** 作为前端开发者，我希望确认整个 UI 改造没有引入新的构建错误或类型错误。

#### 验收标准

1. THE UI SHALL 通过 `pnpm run build` 构建成功
2. THE UI SHALL 通过 `node --run check` 类型检查，不引入新的 TypeScript 错误（允许保留改造前已存在的历史错误）
3. THE UI SHALL 通过现有测试套件（`pnpm run test`），不引入新的测试失败

### 需求 6：无障碍回归

**用户故事：** 作为使用辅助技术的用户，我希望 UI 改造没有破坏现有的无障碍支持。

#### 验收标准

1. THE UI SHALL 保持所有 `aria-label`、`aria-current`、`aria-expanded`、`role` 属性的正确性
2. THE UI SHALL 保持键盘导航在侧边栏、任务卡片、发起面板中的可用性
3. THE UI SHALL 确保新增组件的颜色对比度满足 WCAG AA 标准（4.5:1 正文文字）
