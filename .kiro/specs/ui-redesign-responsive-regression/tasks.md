# 任务清单：响应式适配与回归收尾

## 说明

本任务清单是 UI 改造项目的最终收尾 spec，依赖前 7 个 spec 全部完成。

## 任务

- [ ] 1. 暖色残留值扫描与清理
  - [ ] 1.1 使用 grep 扫描 `client/src/` 中的暖色硬编码值（#4a3727、#7d6856、#c98257、rgba(88,61,39,…) 等）
  - [ ] 1.2 排除 LoadingScreen.tsx、测试文件、.kiro/ 目录和 three/ 材质色值
  - [ ] 1.3 将发现的残留暖色值替换为对应的冷灰值或 CSS 变量引用
  - [ ] 1.4 记录所有替换位置和原因

- [ ] 2. 桌面端断点回归
  - [ ] 2.1 在 1280px 宽度下验证：侧边栏 + 中央内容区 + 右侧面板布局正确
  - [ ] 2.2 在 1440px 宽度下验证：中央内容区自然扩展，无溢出
  - [ ] 2.3 在 1728px 宽度下验证：无水平溢出，卡片间距一致
  - [ ] 2.4 验证侧边栏折叠/展开过渡动画平滑

- [ ] 3. 平板端断点回归
  - [ ] 3.1 在 1024px 宽度下验证：图标侧边栏 + 内容区 + 右侧面板
  - [ ] 3.2 在 768px 宽度下验证：右侧面板折叠或隐藏处理
  - [ ] 3.3 验证所有交互功能在平板端可用

- [ ] 4. 移动端断点回归
  - [ ] 4.1 在 375px 宽度下验证：无侧边栏、底部标签栏、全宽内容
  - [ ] 4.2 在 414px 宽度下验证
  - [ ] 4.3 验证 safe-area-inset-bottom 适配
  - [ ] 4.4 验证核心功能（导航、任务查看、任务发起）在移动端可用

- [ ] 5. 构建与类型检查
  - [ ] 5.1 运行 `pnpm run build` 验证构建成功
  - [ ] 5.2 运行 `node --run check` 验证不引入新的 TypeScript 错误
  - [ ] 5.3 运行 `pnpm run test` 验证不引入新的测试失败

- [ ] 6. 无障碍回归
  - [ ] 6.1 验证侧边栏的 aria-label、aria-current、aria-expanded 属性正确
  - [ ] 6.2 验证发起面板的 role="dialog"、aria-modal、焦点陷阱正常
  - [ ] 6.3 验证任务卡片的键盘导航可用
  - [ ] 6.4 使用浏览器开发者工具检查新增组件的颜色对比度

- [ ] 7. 文档同步
  - [ ] 7.1 更新 `.kiro/steering/project-overview.md` 中的 UI 架构描述
  - [ ] 7.2 更新 README.md 中的截图或界面描述（如有）
