## 当前状态边界

虽然第 4 项已经可以勾选，但这条链路的能力边界仍要写清楚：

- 当前 timeout 是执行器侧的 Promise race 软超时
  - 它会让调用方尽快返回错误
  - 但不等于底层 agent / internal_api 一定被真正取消
- 当前 retry 是固定次数的立即重试
  - 没有指数退避
  - 没有按错误类型差异化重试
- 当前 fallback 只支持一个显式后备目标
  - 还不是多级 fallback 链
- 当前恢复策略主要由 `metadata` 驱动
  - 还没有独立的强类型 request 字段与外部文档接口契约

这些限制不影响“任务已完成”的判断，但影响后续扩展路线。
