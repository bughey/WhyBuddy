# 需求文档：Web-AIGC 平台任务投影

## 目标

让 `web-aigc` 编排实例能自然投影到 Cube 的 `workflow / mission / task` 结构中。

## 需求

### 需求 1：身份关联

每次编排执行都应能关联 `workflowId`、`missionId`、必要时关联 `sessionId`。

### 需求 2：运营连续性

编排执行应复用 Cube 已有的取消、重试、人工决策、产物查看等操作入口。

### 需求 3：回放连续性

编排图执行过程应能映射到当前任务事件流和回放界面。

### 需求 4：报告连续性

最终报告中应能体现节点执行路径、关键输入输出与异常点。
