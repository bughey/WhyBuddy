# UE5 与 Three.js 场景对比指南

## 概述

本文档定义 UE5 场景与 Three.js OfficeRoom 场景的截图对比方法，包括相机位置、对比维度、验收标准与操作流程。目标是确保 UE5 场景在布局、比例、色彩和氛围上与 Three.js 版本保持一致。

---

## 1. 对比目标

### 1.1 核心验收标准

| 维度 | 标准 | 说明 |
|------|------|------|
| 布局准确性 | 偏差 ≤ ±5% | 家具相对位置与 Three.js 版本的偏差 |
| 比例一致性 | 偏差 ≤ ±3% | 家具尺寸比例与房间比例 |
| 色彩匹配度 | 主色调一致 | 墙面、地板、家具主色调视觉一致 |
| 灯光氛围 | 整体感受一致 | 光影方向、明暗对比、色温感受 |
| 空间感 | 视觉一致 | 房间开阔感、区域划分清晰度 |

### 1.2 允许的差异

以下差异属于预期范围，不视为不合格：

- UE5 使用 Lumen GI 产生的更真实光影效果
- UE5 材质的 PBR 物理反射与 Three.js 简化着色的差异
- UE5 后处理效果（Bloom、Vignette、色调映射）带来的视觉增强
- Kenney 模型顶点色与 Three.js 程序化颜色的微小色差
- UE5 阴影精度高于 Three.js 的差异

## 2. 截图规范

### 2.1 分辨率与格式

| 项目 | 要求 |
|------|------|
| 分辨率 | 1920 × 1080（16:9） |
| 格式 | PNG（无损） |
| 色彩空间 | sRGB |
| 后处理 | 保持默认（不额外调色） |
| 命名格式 | `<引擎>_<视角>_<日期>.png` |

**命名示例：**
- `UE5_Overview_20260428.png`
- `ThreeJS_Overview_20260428.png`
- `UE5_WorkArea_20260428.png`

### 2.2 UE5 截图方法

```
# 方法 1：高分辨率截图命令
HighResShot 1920x1080

# 方法 2：使用 Movie Render Queue（更精确）
1. 打开 Movie Render Queue
2. 设置输出分辨率 1920×1080
3. 设置输出格式 PNG
4. 关闭运动模糊
5. 渲染单帧

# 方法 3：编辑器视口截图
1. 在编辑器视口中调整相机
2. 使用 Ctrl+Shift+E 截取视口
```

### 2.3 Three.js 截图方法

```javascript
// 在浏览器控制台中执行
const canvas = document.querySelector('canvas');
const link = document.createElement('a');
link.download = 'ThreeJS_Overview.png';
link.href = canvas.toDataURL('image/png');
link.click();
```

或使用浏览器开发者工具截取 Canvas 元素。

- 家具密度和间距

## 4. 对比维度详解

### 4.1 布局准确性检查

**方法：** 在两张截图上叠加网格线，对比关键物体的相对位置。

| 检查项 | 测量方法 | 合格标准 |
|--------|----------|----------|
| 工位区中心偏移 | 测量工位区几何中心与房间中心的距离比 | ≤ 5% |
| 会议桌位置 | 测量会议桌中心与房间边界的距离比 | ≤ 5% |
| 休息区位置 | 测量沙发中心与房间角落的距离比 | ≤ 5% |
| 家具间距 | 测量相邻家具之间的间距比 | ≤ 5% |
| 墙面距离 | 测量家具与最近墙面的距离比 | ≤ 5% |

### 4.2 色彩匹配检查

**方法：** 使用取色器对比关键区域的颜色值。

| 检查区域 | Three.js 参考色 | 允许偏差 |
|----------|----------------|----------|
| 后墙 | #D8C8B7 | ΔE ≤ 10 |
| 侧墙 | #D2C2B2 | ΔE ≤ 10 |
| 外层地板 | #CBB596 | ΔE ≤ 10 |
| 中层地板 | #D8C2A5 | ΔE ≤ 10 |
| 踢脚线 | #B39C83 | ΔE ≤ 10 |

> ΔE 为 CIEDE2000 色差公式计算值。ΔE ≤ 5 为几乎无法察觉的差异，ΔE ≤ 10 为可接受的微小差异。

### 4.3 灯光氛围检查

**方法：** 主观评估 + 亮度直方图对比。

| 检查项 | 评估方法 | 合格标准 |
|--------|----------|----------|
| 主光方向 | 观察阴影投射方向 | 方向一致 |
| 明暗对比 | 对比亮部/暗部比例 | 整体感受一致 |
| 色温感受 | 对比整体冷暖倾向 | 同为暖白或冷白 |
| 阴影柔和度 | 观察阴影边缘 | UE5 可更柔和（Lumen 特性） |
| 环境光 | 对比暗角和间接光 | 整体亮度感受一致 |

### 4.4 比例一致性检查

**方法：** 测量关键物体在画面中的像素占比。

| 检查项 | 测量方法 | 合格标准 |
|--------|----------|----------|
| 桌面宽高比 | 测量桌面在画面中的宽高像素比 | ≤ 3% 偏差 |
| 椅子高度 | 测量椅子相对桌面的高度比 | ≤ 3% 偏差 |
| 门框高度 | 测量门框相对墙面的高度比 | ≤ 3% 偏差 |
| 房间纵深 | 测量房间纵深方向的透视收缩比 | ≤ 5% 偏差 |

## 5. 对比操作流程

### 5.1 准备阶段

1. 确认 Three.js 场景版本（记录 commit hash）
2. 确认 UE5 场景版本（记录 commit hash）
3. 准备截图工具和取色器
4. 创建截图输出目录：`ue5/screenshots/comparison/`

### 5.2 截图阶段

1. 在 Three.js 中依次设置 6 个相机位置，截取 6 张截图
2. 在 UE5 中依次设置相同的 6 个相机位置，截取 6 张截图
3. 在 UE5 中额外截取 4 种灯光预设的全景截图
4. 按命名规范保存所有截图

### 5.3 对比阶段

1. 将同一视角的 UE5 和 Three.js 截图并排放置
2. 逐项检查布局准确性（叠加网格线）
3. 逐项检查色彩匹配（取色器对比）
4. 逐项检查灯光氛围（主观评估）
5. 逐项检查比例一致性（像素测量）

### 5.4 记录阶段

创建对比报告，格式如下：

```markdown
# 场景对比报告

## 基本信息
- Three.js 版本：<commit hash>
- UE5 版本：<commit hash>
- 对比日期：YYYY-MM-DD
- 对比人：<姓名>

## 全景俯瞰对比
- 布局准确性：✅ / ❌（偏差 X%）
- 色彩匹配：✅ / ❌
- 灯光氛围：✅ / ❌
- 备注：<具体问题描述>

## 工位区对比
...

## 总结
- 通过项：X / Y
- 需修正项：<列表>
- 整体评价：<通过 / 需修正>
```

## 6. 常见问题与解决方案

### 6.1 颜色偏差过大

**原因：** UE5 使用线性色彩空间，Three.js 使用 sRGB。

**解决：**
- 确认 UE5 材质中的颜色值已从 sRGB 转换为线性空间
- 确认 Post Process Volume 的色调映射设置合理
- 确认截图输出为 sRGB 色彩空间

### 6.2 物体比例不一致

**原因：** 坐标系转换错误或单位不一致。

**解决：**
- 确认坐标转换公式：UE5_X = ThreeJS_X × 100, UE5_Y = ThreeJS_Z × -100, UE5_Z = ThreeJS_Y × 100
- 确认 UE5 单位为厘米（cm）
- 确认 Three.js 单位为米（m）

### 6.3 阴影方向不一致

**原因：** Directional Light 角度设置不同。

**解决：**
- 参考 `ue5/docs/lighting-system.md` 中的 Directional Light 参数
- 确认 Pitch 和 Yaw 角度与 Three.js 光源方向对应

### 6.4 FOV 不匹配

**原因：** 相机视场角设置不同。

**解决：**
- 确认两端使用相同的 FOV 值
- Three.js 默认 FOV 为垂直方向，UE5 默认为水平方向，需要换算
